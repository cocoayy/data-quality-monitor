# Data Quality Monitor

## 概要

本プロジェクトは、オープンデータの品質問題を可視化し、改善につなげることを目的としたデータ品質監視ツールです。

近年、政府・自治体・公共機関においてオープンデータの公開が進んでいますが、以下のような課題が存在します。

* 更新が停止している
* メタデータが不完全
* リンク切れやAPI障害
* フォーマットの不統一

本ツールは、これらの課題を「品質スコア」として定量化し、継続的に監視・改善できる仕組みを提供します。

---

## 背景

オープンデータは公開されているだけでは価値を持たず、「正しく使える状態」であることが重要です。

しかし実際には、

* 古いデータが更新されない
* ダウンロードできない
* 構造がバラバラ

といった理由で活用されないケースが多く存在します。

本プロジェクトでは、これらの問題を技術的に解決し、データの信頼性（Data Reliability）を向上させることを目指しています。

---

## 主な機能

### 1. データ品質スコア

データセットごとに以下の指標でスコアを算出します。

* Completeness（完全性）
* Freshness（更新性）
* Accessibility（アクセス性）
* Format Quality（形式品質）

総合スコア（0〜100）として可視化します。

---

### 2. ダッシュボード

* データ品質の全体傾向を表示
* ランク分布（A〜E）
* アラート件数
* 平均スコア

---

### 3. データセット一覧・詳細

* フィルタ・ソートによる検索
* 個別データの品質確認
* スコアの内訳と理由表示

---

### 4. スコア理由（Explainability）

スコアの根拠を明示することで、改善ポイントを特定可能にします。

例：

* 「ライセンスが未設定」
* 「更新が15日遅延」
* 「APIが応答していない」

---

### 5. アラート機能

* スコア低下の検知
* 更新停止の検知
* アクセス不可の検知

## アーキテクチャ

```text
Collector → Queue → Processor → DB → API → Frontend
```

* Collector: データ収集
* Processor: スコア計算
* DB: スコア・履歴保存
* API: データ提供
* Frontend: 可視化
